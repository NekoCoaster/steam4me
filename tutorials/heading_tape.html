Error Message - Failed To Initialize Heading Tape
by Yuri Sos

Overview

Ah, you've got to love MSTS's obscure error messages! Here's another one for you to ponder:

Of course, you look at this message and it instantly points to the source of the error - not! It appears as you start an activity (or "Explore") a route, sometimes a route that you've run successfully before, but never in winter before. And therein lies a clue to this error message's cause.

 Return to top of page

The Cause of the Error

This error message is caused by a missing or corrupt "ACleanTrack1.ace" and / or "ACleanTrack2.ace". These files provide the texture for the sleepers and ballast for your track in the sim. These two files are normally found in both the Route_Name\Textures and the Route_Name\Textures\Snow folder: not necessarily exactly the same as the Snow ACE files usually portray a snow-covered track.

You will usually see this error message before the one above as the route loads:

If you click "Yes", the "Failed to initialize heading tape" immediately appears.

 Return to top of page

Solution

One or both files are missing. Check both the Route_Name\Textures and the Route_Name\Textures\Snow folders: you will most likely find that either "ACleanTrack1.ace" and/or "ACleanTrack2.ace" are missing. Replace them and the route will work fine. Note: if you don't use the season containing the missing ACE files, you won't see the error message: ie

  • If I remove "ACleanTrack1.ace" and/or "ACleanTrack2.ace" from the Route_Name\Textures folder (but leave them in the \SNOW folder) and run an activity utilising a snow season, the route runs ok.
  • If I remove "ACleanTrack1.ace" and/or "ACleanTrack2.ace" from the Route_Name\Textures\SNOW folder (but leave them in the \Textures folder) and run an activity utilising a non-snow season, the route runs ok.

    • If by some chance, all four files (two in Textures, two in Textures\Snow) are present, they are corrupt. Replace them with ACE files from a known good route.

    Hope this helps.
